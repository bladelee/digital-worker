### Mattermost 插件 UI 调试

**问题:** 插件 Webapp UI 不显示在 Mattermost 中

**调试过程:**
1. 发现 `has_webapp: false` - Mattermost 未检测到 webapp
2. 分析官方 GitHub 插件的打包结构
3. **修复打包结构** - 文件必须放在 `dist/{plugin-id}/` 子目录中

**正确的打包结构:**
```
com.openclaw.digital-worker/
├── plugin.json
├── server/
│   └── dist/
│       └── plugin-linux-amd64
└── webapp/
    └── dist/
        └── main.js
```

**修复后:**
- `has_webapp: true` ✅
- Webapp bundle 被 Mattermost 识别

**遗留问题:**
- Webapp JavaScript 未执行（没有控制台日志）
- 401 Unauthorized 和 501 Not Implemented 错误
- 可能是 Mattermost 11.x 的 Webapp SDK 兼容性问题

**关键发现:**
- `registerMainMenuAction` 需要 3 个参数: (text, action, mobileIcon)
- `registerChannelHeaderIcon` 只接受 1 个参数: (component)
- 官方插件使用 `registerRightHandSidebarComponent` 作为主要 UI 入口

**打包命令:**
```bash
mkdir -p dist/package/com.openclaw.digital-worker/webapp/dist
mkdir -p dist/package/com.openclaw.digital-worker/server/dist
cp plugin.json dist/package/com.openclaw.digital-worker/
cp server/dist/plugin-linux-amd64 dist/package/com.openclaw.digital-worker/server/dist/
cp webapp/dist/main.js dist/package/com.openclaw.digital-worker/webapp/dist/
cd dist/package && tar -czvf ../../dw.tar.gz com.openclaw.digital-worker
```

## 深夜工作 - 插件 UI 完全修复 (22:00-23:00)

### 问题解决记录

#### 问题 1: `require is not defined`
- **原因:** webpack externals 配置 `react: 'window.React'` 生成了 `require("window.React")`
- **解决:** 改为 `react: 'React'`

#### 问题 2: 插件菜单项点击无反应
- **原因:** action 回调没有执行任何操作
- **解决:** 点击时 dispatch `showRHSAction` 打开右侧边栏

#### 问题 3: API `invalid input syntax for type uuid`
- **原因:** Mattermost team ID 是字符串，数据库期望 UUID
- **解决:** 修改 `listWorkers` 支持 mattermost_team_id 查询

#### 问题 4: 创建员工 500 错误
- **原因:** 
  1. Docker 容器 `MATTERMOST_URL=localhost:8065` 无法访问宿主机
  2. `tenantId: 'auto'` 不是有效 UUID
- **解决:**
  1. 改为 `MATTERMOST_URL=http://host.docker.internal:8065`
  2. 后端支持 `tenantId: 'auto'` 自动获取默认用户

### 关键经验总结

1. **webpack externals 必须用 `'React'` 不是 `'window.React'`**
2. **插件打包结构必须包含正确的目录层级**
3. **Mattermost 11.x 使用 `window.store` 不是 `__REDUX_STORE__`**
4. **Docker 容器访问宿主机用 `host.docker.internal`**
5. **API 需要 bridge Mattermost ID 和平台 UUID**
