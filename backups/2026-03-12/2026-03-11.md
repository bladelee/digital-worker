## Mattermost 群聊消息接收分析

### 问题描述
Mattermost 服务端不会向 Bot 用户推送不提及 bot 的普通群聊消息。

### 结论
这是 Mattermost 的设计决策，不是 OpenClaw 的问题：
- Bot 只能收到 DM 和 @提及的消息
- 普通群聊消息不会被推送给 Bot

### 解决方案
开发 Mattermost 服务端插件 `mattermost-channel-forwarder`，使用 `PostHasBeenPublished` 钩子将群聊消息转发给 Bot 用户。

### 插件开发进展
- 插件目录: `~/.openclaw/extensions/mattermost-channel-forwarder/`
- 主要文件: `main.go`, `manifest.json`, `plugin.json`
- 功能: 监听 `PostHasBeenPublished` 事件，将公开/私有频道的消息通过 WebSocket 事件 `bot_channel_message` 转发给频道中的 Bot 用户

### OpenClaw 端改动
- 文件: `extensions/mattermost/src/mattermost/monitor-websocket.ts`
- 添加了 `bot_channel_message` 事件处理
- Patch 文件: `~/.openclaw/extensions/mattermost-channel-forwarder/openclaw-mattermost.patch`

### Mattermost 配置修改
- 启用了插件上传: `PluginSettings.EnableUploads = true`
- 插件已安装并激活: `com.openclaw.bot-channel-forwarder v1.0.1`

### 已验证
- ✅ 插件的 `MessageHasBeenPosted` 钩子被正确调用（注意：不是 `PostHasBeenPublished`）
- ✅ 插件成功检测到频道中的 Bot 用户
- ✅ 插件成功调用 `PublishWebSocketEvent` 广播消息
- ❌ 客户端（无论是 Bot token 还是普通用户 token）无法收到 `bot_channel_message` 事件

### 根本原因
**Mattermost 的 `PublishWebSocketEvent` API 会将事件名修改为 `custom_<plugin_id>_<event_name>`**
- 我们发送的事件名: `bot_channel_message`
- 实际的事件名: `custom_com.openclaw.bot-channel-forwarder_bot_channel_message`
- 客户端监听的是错误的事件名

### 下一步
1. 验证完整事件名是否能被接收
2. 检查 `EnableWebHubChannelIteration` 配置是否启用

### 关键发现

#### 消息分发架构
1. **Hub 系统** (`server/channels/app/platform/web_hub.go`)
   - 管理所有 WebSocket 连接
   - 支持按 UserId、ChannelId、TeamId 过滤
   - 使用通道实现异步消息传递

2. **PublishSkipClusterSend** (`server/channels/app/platform/cluster.go`)
   - 只处理 `UserId`，不直接处理 `ChannelId`
   - `ChannelId` 在 Hub 内部处理

3. **ShouldSendEvent** (`server/channels/app/platform/web_conn.go`)
   - 检查用户是否是频道成员
   - 过滤不符合条件的消息

#### 插件事件名机制
```go
func (ps *PlatformService) PublishWebSocketEvent(productID, event string, ...) {
    ev := model.NewWebSocketEvent(
        model.WebsocketEventType(fmt.Sprintf("custom_%v_%v", productID, event)),
        ...
    )
}
```
